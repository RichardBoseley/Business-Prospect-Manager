# outreach by smallteam.ai

Local-business outreach and lightweight CRM for venue and service businesses.
First customer workspace: Game On Entertainment Centre, Cranbourne West VIC.

Next.js (App Router) + React + Tailwind v4, recreated pixel-faithfully from
the design handoff in `design_handoff_outreach_crm/`. Runs on a single EC2
instance with SQLite; multi-tenant from day one.

## Environment layout

| Piece | What it is |
|---|---|
| EC2 | One `t4g.small` Ubuntu 24.04 ARM instance in `ap-southeast-2`, 20 GB gp3, Elastic IP. No load balancer, no RDS, no NAT. |
| App process | PM2 service `bpm` running `npm start` (Next.js on :3000) as `ubuntu`, survives reboot. |
| Web | nginx on the instance: 80 redirects to 443 (self-signed certificate), 443 proxies to :3000. |
| Database | SQLite at `/var/app/data/app.db` — outside the repo checkout, auto-created and seeded on first run. |
| Backups | Nightly cron (03:00 UTC) copies the DB to a private, versioned S3 bucket `business-prospect-manager-backups-<account>`; backups expire after 30 days. |
| Auth | Cognito user pool `smallteam-apps` (shared across smallteam apps) + an app client for this project. Hosted UI, admin-created users only, no self-signup. |
| Secrets | Runtime env (Cognito client secret etc.) lives in SSM Parameter Store `/business-prospect-manager/env` and is written to `.env` on the instance — never in the repo. |
| CI/CD | GitHub Actions: merge to `main` → SSH to the instance → `infra/deploy.sh`. |
| Cost control | AWS Budgets alert at US$25/month. |

Everything AWS-side is tagged `project=business-prospect-manager` and managed
by the idempotent scripts in `infra/`.

## Run locally

```bash
npm install
npm run build   # or: npm run dev
npm start
```

No env vars needed: the SQLite file is created and seeded automatically at
`./.data/app.db` (gitignored), and auth is disabled when Cognito env vars are
absent. Open http://localhost:3000.

## Repository structure

```
src/
  app/            routes (one per screen) + API routes (bootstrap, auth,
                  dnc, unsubscribe, health)
  components/     app shell, sidebar, lead drawer, wizard, screens
  lib/
    db/           SQLite schema, seeding, repository (server-only)
    services/     stubbed integrations — discovery, drafting, send/track,
                  unsubscribe, maps — behind interfaces so real providers
                  drop in without touching UI code
    data/seed.ts  seed data (from the design prototype)
    store.tsx     client state, hydrated from /api/bootstrap
    auth.ts       Cognito hosted-UI config (env-driven)
  middleware.ts   requires a valid Cognito session when auth is configured
infra/            provision.sh / deploy.sh / teardown.sh (bash + AWS CLI)
.github/workflows/deploy.yml
```

## Deploys

Work on branches, open a PR, merge to `main`. On merge, the `Deploy` workflow
SSHes to the instance and runs `infra/deploy.sh` (git pull, `npm ci`, build,
PM2 restart; the DB migrates/seeds itself on boot).

Required repo secrets (Settings → Secrets and variables → Actions):

- `EC2_HOST` — the Elastic IP
- `EC2_USER` — `ubuntu`
- `EC2_SSH_KEY` — contents of the deploy key generated by `provision.sh`
  (also retrievable: `aws ssm get-parameter --name /business-prospect-manager/deploy-key --with-decryption --query Parameter.Value --output text`)

Manual deploy: `ssh ubuntu@<elastic-ip> 'bash /var/app/business-prospect-manager/infra/deploy.sh'`.

## Access

- URL: `https://<elastic-ip>` — the certificate is self-signed, so accept the
  browser warning (or put a real domain + certificate in front later).
- Login: Cognito hosted UI appears automatically. Users are created by an
  admin only (`aws cognito-idp admin-create-user --user-pool-id <pool> ...`);
  there is no self-signup. First login forces a password change.
- Sign out: link at the bottom of the sidebar, or `/api/auth/logout`.

## Provision / teardown

```bash
cd infra
./provision.sh <ssh-cidr> [budget-email]   # idempotent; ~5 min to first boot
./teardown.sh                              # keeps backups, Cognito, budget
./teardown.sh --delete-backups --delete-cognito-client --delete-budget
```

Notes:

- `<ssh-cidr>` restricts port 22. GitHub-hosted runners need SSH for CI
  deploys and have no fixed range — use `0.0.0.0/0` (key-only auth) if you
  use the workflow, or restrict it and deploy manually.
- The billing alert email defaults to `richard@smallteam.ai` — change it in
  `infra/provision.sh` arguments or AWS console → Billing → Budgets.
- The Cognito pool `smallteam-apps` is shared by future apps: teardown never
  deletes the pool, its domain or its users.

## Restore a DB backup

```bash
aws s3 ls s3://business-prospect-manager-backups-<account>/backups/
ssh ubuntu@<elastic-ip>
  pm2 stop bpm
  aws s3 cp s3://business-prospect-manager-backups-<account>/backups/app-<date>.db /var/app/data/app.db
  rm -f /var/app/data/app.db-wal /var/app/data/app.db-shm
  pm2 start bpm
```

To start over from seed data instead: stop the app, delete
`/var/app/data/app.db*`, start the app — it recreates and reseeds itself.

## External services

Email send/track, unsubscribe, business discovery, AI drafting and maps are
stubbed behind service interfaces in `src/lib/services/` returning seed data.
Each has a mock implementation and a typed interface; a real provider is a
drop-in replacement with no UI changes. The one-click unsubscribe endpoint
(`GET /api/unsubscribe?b=<businessId>`) is live and writes a permanent DNC
entry (Spam Act 2003 — entries cannot be deleted from the app).
